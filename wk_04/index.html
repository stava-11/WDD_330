Notes from reading

  • Chapter 8 - Forms
    • Can access form controls by their name attribute
      • const input = form.searchInput
    • autofocus attribute gives focus to this element when a page loads
    • disable attribute will disable a button and can be changed by setting disabled equal to false or true by using javascript
  • Chapter 12 - OOP
    • Encapsulation: the inner workings are kept hidden inside the object and only the essential functionalities are exposed to the end user
    • Polymorphism: the same process can be used for different objects
    • Inheritance: taking the features of one object then adding some new features
    • Classes: before ES6 this did not exist in javascript. It used prototype-based design and still uses prototypal inheritance model in the background
    • Variable scope is used to keep properties and methods private inside of a class declaration
    • enumerable properties of an object will not show up in a for-in loop
  • Chapter 15 - Modular Javascript
    • jQuery is the most popular of all the Javascript libraries
      • Useful with old browsers
    • libraries
      • Advantages
        • Very well tested and optimized
        • Lots of documentation
      • Disadvantages
        • Have to include the code which increases the size of your program
        • Implementation is not exactly how you need it
    • Modular Javascript
      • A self-contained piece of code that provides functions and methods that can then be used in other files and by other modules
      • Coupling of code refers to how dependent certain elements or modules of code are on each other. Two pieces of code are said to be tightly coupled if one relies on the other to run.
      • Important points about modules
        • All code in modules is always in strict mode without the need for 'use strict' and there is no way to opt out of this.
        • A module has its own global scope, so any variables created in the top-level of a module can only be accessed within that module.
        • The value of this in the top level of a module is undefined , rather than the global object.
        • You can't use HTML-style comments in modules (although this isn't very common in any JavaScript program these days).
