MDN: Validating Forms

  • Client Side validation is done in the browser and server-side validation is done on the server.
  • 3 Main Reasons for validation
    • We want to get the right data, in the right format
    • We want to protect our users' data
    • We want to protect ourselves
  • WARNING: Never trust passed to your server from the client. Even if your form is validating correctly and preventing malformed input on the client-side, a malicious user can still alter the network request.
  • Client-side Validation
    • Built-in form validation uses HTML5 form validation and has better performance than Javascript, but is not as customizable as Javascript.
      • 'required' attribute
      • 'pattern' attribute
      • Constrain the length of the entries (minlength, maxlength)
      • Constrain value of entries (number)
    • Javascript validation
      • The Constraint Validation API
        • validationMessage: returns a localized message describing the validation constraints that the control doesn't satisfy.
        • validity: returns ValidityState object that contains several properties describing the validity state of the element.
        • willValidate: returns true if the element will be validated when the form is submitted, false otherwise.
